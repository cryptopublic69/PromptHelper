# 更新日志

## 未发布 - 2026-07-16

### 提示词浏览与排序

- 提示词列表新增 Pointer Events 拖放排序，沿用资料库目录的 `grab` / `grabbing` 光标、移动阈值、源项目淡出和前后插入位置提示。
- 搜索结果保持只读顺序，拖放排序仅在普通分类视图且存在多条提示词时启用。
- 移除提示词逐条上移、下移按钮，保留置顶操作并以直接拖动简化排序流程。
- 新增提示词完整内容查看器，可查看未截断正文、字符数和所属类型／分类，并可直接复制完整内容。
- 完整内容查看器新增编辑入口，可从详情直接切换到提示词编辑器。

### 记录结构与 Pin

- 提示词记录新增稳定 ID、创建时间、更新时间、显式排序值和可选 Pin 状态；新建与编辑记录会写入真实时间，保存时同步当前分类内的排序值。
- 旧版纯字符串和缺少元数据的对象会在首次打开后自动升级；历史记录时间保留为 `null`，不会伪造创建或更新时间，并清理遗留的 `_category_order` 字段。
- 新增 Pin／取消 Pin 操作，Pin 项目固定显示在分类顶部；置顶和拖放分别在 Pin 分组或普通分组内生效，避免跨分组破坏顺序。
- 跨分类移动会保留 Pin 状态，并将记录插入目标分类的对应分组；新建提示词默认加入普通分组顶部。

### 操作菜单与界面细节

- 提示词默认只显示查看和三点入口，点击三点后再展开查看、编辑、置顶、移动和删除操作。
- 提示词及资料库目录的二级操作菜单会在鼠标离开当前项目时自动收起，避免菜单状态残留。
- 新建提示词编辑器与编辑模式的数据来源明确分离，确保新增时不会继承现有提示词内容。
- 精简加密资料库解锁页面说明文字，使密码输入区域更紧凑。
- 将提示词移动目标从下拉框改为可展开的类型／分类树，显示可用位置、分类记录数、当前位置和已选目标。
- 修正导入与导出按钮的图标方向，并将浏览器标题、站点图标、侧栏品牌标识和解锁页标识统一为 PromptHelper 应用信息。

### 外观设置

- 新增暗黑与白色主题，可同步切换工作台、弹窗和解锁页面，并在本机保存选择。
- 新增 14–20px 全局字号滑杆、四档预设、实时预览和恢复默认操作；界面文字改用相对单位以跟随全局字号缩放。
- 补充白色主题下的资料树、提示词卡片、操作菜单、移动树、设置面板、滚动条和反馈状态样式。

### 验证

- `cargo test --manifest-path src-tauri/Cargo.toml`

## 5.0.0 - 2026-07-16

### 全新架构

- 将旧版 Tkinter PromptHelper 重构为 Tauri 2、React 19、TypeScript 和 Rust 桌面应用。
- 保留本地 JSON 资料库模式，不依赖云端服务或 AI 模型。
- 提供 NSIS Windows 打包配置、Tauri 权限配置和完整应用图标资源。

### 界面与工作区

- 使用无系统边框窗口和自定义最小化、最大化、关闭按钮。
- 顶栏直接显示工作标签，支持新建、关闭、双击重命名和拖动排序。
- 标签超出可见范围后支持高速度平滑滚轮横向滚动，新建标签始终保持可见。
- 新建标签默认保持空白，不自动选择资料库内容。
- 每个标签独立保存当前类型、分类、搜索词、展开状态和自定义名称，并在重启后恢复。

### 资料库导航

- 左侧导航改为类型与分类组成的树形目录，并显示类型、分类和提示词数量。
- 一级目录展开后以分组背景和选中状态突出层级关系。
- 类型与分类悬停时显示三点入口，点击后才展开对应的新增、重命名和删除操作。
- 类型和分类均支持 Pointer Events 拖放排序，使用 `grab` / `grabbing` 光标、移动阈值和插入位置提示。
- 分类顺序通过 JSON 对象键顺序持久化，并由 Rust `serde_json` 保序写回，避免增加旧版不认识的元数据字段。
- 左侧资料库滚动条贴合侧栏边缘，同时保留目录内容留白。

### 提示词管理

- 支持类型、分类和提示词的新增、编辑、重命名与删除。
- 提示词支持可选标题和正文，并兼容旧版纯字符串记录。
- 支持一键复制、置顶、上移、下移、跨分类移动和右键快捷菜单。
- 支持跨全部类型和分类搜索标题及正文，并显示搜索结果来源。
- 删除类型、分类或提示词前提供确认提示，所有修改均保存到本地资料库。

### 导入、导出与数据安全

- 支持明文 JSON 的合并导入和覆盖导入，并统计新增类型、分类和提示词数量。
- 支持将当前资料库导出为格式化明文 JSON。
- 兼容旧版 V4 的 `ENC:` 加密文件、PBKDF2-SHA256 参数和数据封装格式。
- 支持启用加密、修改密码、关闭加密，以及中文和 Unicode 数据往返保存。
- 保存时使用临时文件、磁盘同步、备份替换和失败恢复，降低写盘中断导致的数据损坏风险。
- Rust 测试覆盖 Python V4 加密样本解密、Unicode 加密往返和错误密码拒绝。

### 开发体验与性能

- 新增 `start-dev.cmd`，双击即可检查 Node.js、Rust 和依赖并启动 Tauri 热更新开发模式。
- 开发脚本通过 `PROMPT_HELPER_DATA_FILE` 与旧版 EXE 共用指定的现有资料库，并在启动失败或退出后保留窗口显示结果。
- Rust Dev 配置启用 `opt-level = 2` 和增量编译，在不改变 480,000 轮 PBKDF2、加密格式或数据库兼容性的前提下显著提升解锁和保存速度。
